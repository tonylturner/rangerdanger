# PDF update prompt

Internal-tooling document. Not part of the public docs site (the
leading-underscore directory is the convention for that).

The workshop handout PDF (`lab_setup_reference.pdf`, kept by the
maintainer locally) is generated by Claude.ai web, not built in this
repo. When the install workflow drifts, paste the prompt below into a
fresh Claude.ai web conversation along with the previous PDF as an
attachment, and Claude web will regenerate the PDF with the
corrections.

Update the corrections list whenever the install flow changes. Treat
this file as the single source of truth for what the PDF should say,
checked against `docs/quickstart.md` and `setup.sh` at PDF-regen time.

## Prompt

Copy everything between the `---` markers below into a Claude.ai web
conversation. Attach the previous version of the PDF in the same
message.

---

I need you to update the attached PDF "Lab Setup" workshop handout for
RangerDanger. The technical content has drifted since you generated it;
the visual design is fine and I want to preserve it. Make these specific
corrections:

1. PREREQUISITES section - keep "RAM 16 GB minimum, 32 GB recommended"
   (this is host RAM, accounting for the Docker VM allocation, macOS
   itself, and the student's browser - measured live against the
   running stack: idle ~4 GB, peak ~6-8 GB, plus host-OS overhead). Add
   a clarifying parenthetical to the RAM line: "(host RAM; 8 GB of it
   must be allocated to the Docker VM)". Disk line currently says
   "~25 GB free for images". Change to "30 GB free minimum, 50 GB
   recommended" - matches the threshold setup.sh warns at.

2. INSTALL -> BEFORE YOU BEGIN - current text says "set Memory to 8 GB or
   more". Keep that, it's correct and is the Docker VM allocation,
   distinct from the host-RAM line above.

3. INSTALL -> OPTION A . ONLINE - currently shows the build-from-source
   command. Replace with the released-image install:

       git clone https://github.com/tonylturner/rangerdanger
       cd rangerdanger
       ./setup.sh

   Subtitle should say "pull pre-built images from GHCR" instead of
   "build images locally from GitHub". Add a small note below the code
   block: "For developers / contributors who want to build from source
   instead, run `docker compose up -d --build` after the clone."

4. INSTALL -> OPTION B . INSTRUCTOR SSD - the manual `docker load` +
   `docker compose -f docker-compose.workshop.yml up -d` pattern is
   wrong. There is no docker-compose.workshop.yml file. Replace the
   entire option with the supported flow:

   For Apple Silicon Mac:
       tar xzf /Volumes/WORKSHOP/rangerdanger.tgz -C ~
       cd ~/rangerdanger
       ./setup.sh --from-tarballs /Volumes/WORKSHOP

   For Intel/AMD64 Mac, Windows, Linux (same command; setup.sh
   auto-detects arch):
       tar xzf /Volumes/WORKSHOP/rangerdanger.tgz -C ~
       cd ~/rangerdanger
       ./setup.sh --from-tarballs /Volumes/WORKSHOP

   For Windows PowerShell specifically:
       tar -xzf D:\WORKSHOP\rangerdanger.tgz -C $HOME
       cd $HOME\rangerdanger
       .\setup.ps1 -FromTarballs D:\WORKSHOP

   Remove the "Then start the workshop stack:" section entirely -
   setup.sh handles compose internally with the offline overlay.

5. SERVICE ENDPOINTS table - add two rows after containd SSH:

       FUXA HMI       http://localhost:8088/apps/fuxa-hmi/    -
       OpenPLC        http://localhost:8088/apps/openplc/     -

6. Add a small new section between PREREQUISITES and INSTALL titled
   "PRE-WORKSHOP CHECK" with one line:

       Run `./setup.sh --check-only` on each laptop the night before
       to verify Docker, Compose v2, ports 8088/9080/9443/2222 free,
       disk, and RAM. No install - just preflight.

   This is critical for workshop morning success and was missing.

7. Add to PREREQUISITES the four loopback ports that must be free:
   "Loopback ports 8088, 9080, 9443, 2222 free".

Please preserve the visual design exactly - layout, colors, the
numbered section badges, fonts, the architecture diagram screenshot,

the code blocks.
